Motivation

Mengapa Prediksi Harga Saham?

Prediksi harga saham adalah salah satu aplikasi time series forecasting yang paling menantang. Datanya bersifat non-stasioner, memiliki noise tinggi, dan dipengaruhi faktor eksternal yang sulit dimodelkan.


Deep learning berbasis RNN, khususnya GRU (Gated Recurrent Unit), terbukti efektif menangkap dependensi temporal jangka panjang dalam data finansial dibanding model statistik klasik seperti ARIMA.


BBCA dipilih sebagai benchmark, yaitu saham perbankan terbesar di Indonesia dengan data historis panjang
OHLCV (Open, High, Low, Close, Volume) sebagai fitur input multi-dimensi
Fokus pada evaluasi arsitektur, bukan strategi trading
Model ini bersifat akademis dan bukan financial advice
GRU vs LSTM
GRU lebih efisien, dengan 2 gate (reset & update) dibanding LSTM yang memakai 3 gate
Training lebih cepat dengan performa comparable pada sequence pendek
Lebih robust terhadap gradient vanishing pada sequence panjang
BBCA Data

3,500+ hari perdagangan BBCA digunakan, dengan train/test split 80/20 dan MinMaxScaler normalization per fitur OHLCV.

MinMaxScaler

Setiap fitur OHLCV dinormalisasi ke [0,1] secara independen. Scaler di-fit hanya pada train set untuk mencegah data leakage.

Sequence Windowing

Sliding window menghasilkan (X, y) pairs: X = lookback sequence, y = harga Close hari berikutnya. Lookback window dioptimalkan per arsitektur.

Inverse Transform

Prediksi model di-inverse transform kembali ke skala harga asli sebelum evaluasi. MSE dan MAE dihitung dalam satuan rupiah.

Custom Architectures

TF-BIGRU & TFGRUGRU, Detail

TF-BIGRU dirancang untuk menggabungkan feature transformation sebelum GRU bidirectional. Ini memungkinkan model mengekstrak representasi yang lebih informatif sebelum masuk ke mekanisme recurrent.

Feature transformation layer mengekstrak representasi awal
Bidirectional GRU memproses forward & backward context
Dense head untuk proyeksi ke prediksi Close price

TFGRUGRU menggunakan dual GRU stack dengan layer transformasi di tengah, sehingga menciptakan representasi hierarkis yang lebih ekspresif.

First GRU layer: low-level temporal pattern extraction
Transformation bridge antar GRU layer
Second GRU layer: higher-order sequence modeling
TF-BIGRU Flow
Input (OHLCV) → Transform → BiGRU → Dense → Close
TFGRUGRU Flow
Input → GRU 1 → Transform → GRU 2 → Dense

Motivasi desain: Transformation layer bertindak sebagai bottleneck yang memaksa model mengkompresi informasi sebelum GRU, sehingga mengurangi noise dan meningkatkan generalisasi.
